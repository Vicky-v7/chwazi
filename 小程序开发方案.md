# Chwazi 小程序开发方案

## 一、注册与准备阶段

### 1.1 注册微信小程序账号

1. 访问 [微信公众平台](https://mp.weixin.qq.com/)，点击"立即注册"，选择"小程序"
2. 填写**未注册过公众号/开放平台的邮箱**，设置密码
3. 邮箱激活后，选择主体类型为**"个人"**
4. 上传身份证正反面照片，完成人脸识别验证
5. 填写管理员信息并微信扫码验证

**费用**：个人开发者注册完全免费（企业认证需 300 元，个人不需要）

**注意**：个人主体小程序无法使用微信支付功能，但本项目不涉及支付，没有影响。

### 1.2 获取 AppID

- 注册完成后，登录小程序管理后台
- 进入「开发管理」->「开发设置」，查看并记录 **AppID**

### 1.3 开发环境搭建

| 工具 | 说明 | 下载地址 |
|------|------|----------|
| **微信开发者工具** | 官方 IDE，用于编码、预览、调试、上传 | https://developers.weixin.qq.com/miniprogram/dev/devtools/stable.html |
| **Node.js** | JavaScript 运行环境 | https://nodejs.org/ |
| **VS Code**（可选） | 代码编辑器，配合开发者工具使用 | https://code.visualstudio.com/ |

### 1.4 备案要求

自 2023 年 9 月起，微信小程序必须完成备案后才能上架。备案流程：
1. 在小程序管理后台提交备案申请
2. 填写主体信息、负责人信息
3. 等待审核通过（通常 1~5 个工作日）

> 建议：开发期间就开始备案流程，避免开发完成后等备案耽误上线。

## 二、技术栈选择

### 推荐方案：微信原生开发

| 对比维度 | 原生开发 | Taro | uni-app |
|---------|---------|------|---------|
| 性能 | 最优，冷启动 ~680ms | 冷启动 ~820ms | 冷启动 ~760ms |
| 包体积 | 最小 | +30~50% | +40~60% |
| Canvas/触控 API | 完全支持 | 需适配层 | 需适配层 |
| 学习成本 | WXML/WXSS/JS | React | Vue |
| 跨端能力 | 仅微信 | 多端 | 多端 |

**选择原生的理由**：
1. **项目体量小**：单页面为主的工具型应用，不需要复杂框架
2. **纯前端无后端**：没有跨端需求（只做微信小程序）
3. **深度依赖 Canvas 和触控 API**：原生直接调用，无中间层损耗，性能最优
4. **减少依赖**：无需额外安装 npm 包，项目更轻量

## 四、核心技术实现方案

### 4.1 Canvas 画布初始化

使用 Canvas 2.0 API，在页面 `onReady` 生命周期中初始化：

```javascript
// pages/index/index.js
onReady() {
  const query = wx.createSelectorQuery()
  query.select('#gameCanvas')
    .fields({ node: true, size: true })
    .exec((res) => {
      const canvas = res[0].node
      const ctx = canvas.getContext('2d')

      // 适配设备像素比，避免模糊
      const dpr = wx.getSystemInfoSync().pixelRatio
      canvas.width = res[0].width * dpr
      canvas.height = res[0].height * dpr
      ctx.scale(dpr, dpr)

      this.canvas = canvas
      this.ctx = ctx
      this.startRenderLoop()
    })
}
```

对应的 WXML 结构：

```xml
<!-- pages/index/index.wxml -->
<canvas
  id="gameCanvas"
  type="2d"
  style="width: 100vw; height: 100vh;"
  bindtouchstart="onTouchStart"
  bindtouchmove="onTouchMove"
  bindtouchend="onTouchEnd"
></canvas>
```

### 4.2 多指触控检测

通过 `e.touches` 数组追踪所有手指的位置和状态：

```javascript
// 存储当前所有手指信息
data: {
  fingers: {},        // { touchId: { x, y, color, radius, active } }
  selectedId: null,   // 被选中的手指ID
  isSelecting: false, // 是否正在选择中
},

onTouchStart(e) {
  const touches = e.changedTouches
  touches.forEach(touch => {
    this.data.fingers[touch.identifier] = {
      x: touch.clientX,
      y: touch.clientY,
      color: this.getRandomColor(),
      radius: 0,       // 初始半径，动画中逐渐变大
      active: true,
    }
  })
  this.checkAndStartSelection()
},

onTouchMove(e) {
  const touches = e.changedTouches
  touches.forEach(touch => {
    if (this.data.fingers[touch.identifier]) {
      this.data.fingers[touch.identifier].x = touch.clientX
      this.data.fingers[touch.identifier].y = touch.clientY
    }
  })
},

onTouchEnd(e) {
  const touches = e.changedTouches
  touches.forEach(touch => {
    delete this.data.fingers[touch.identifier]
  })
  // 如果有手指提前离开，取消选择
  this.cancelSelection()
},
```

### 4.3 随机选择算法

```javascript
// utils/random.js

/**
 * 从手指列表中随机选择指定数量
 * @param {Object} fingers - 手指对象 { id: fingerInfo }
 * @param {Number} count - 需要选中的数量
 * @returns {Array} 选中的手指ID数组
 */
function randomSelect(fingers, count = 1) {
  const ids = Object.keys(fingers)
  if (ids.length <= count) return ids

  // Fisher-Yates 洗牌算法
  const shuffled = [...ids]
  for (let i = shuffled.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1))
    ;[shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]]
  }

  return shuffled.slice(0, count)
}

module.exports = { randomSelect }
```

### 4.4 动画渲染循环

```javascript
startRenderLoop() {
  const render = () => {
    const ctx = this.ctx
    const { width, height } = this.canvas

    // 清空画布
    ctx.clearRect(0, 0, width, height)

    // 绘制每个手指的圆圈
    Object.keys(this.data.fingers).forEach(id => {
      const finger = this.data.fingers[id]
      if (!finger.active) return

      // 圆圈渐入动画
      if (finger.radius < 40) {
        finger.radius += 2
      }

      ctx.beginPath()
      ctx.arc(finger.x, finger.y, finger.radius, 0, Math.PI * 2)
      ctx.fillStyle = finger.color

      // 选中的手指：更大、更亮
      if (this.data.selectedId === id) {
        ctx.globalAlpha = 1.0
        finger.radius = 60
      } else if (this.data.selectedId) {
        // 未选中的：逐渐淡出
        ctx.globalAlpha = Math.max(0, ctx.globalAlpha - 0.02)
      } else {
        ctx.globalAlpha = 0.7
      }

      ctx.fill()
      ctx.closePath()
      ctx.globalAlpha = 1.0
    })

    this.animationId = this.canvas.requestAnimationFrame(render)
  }
  render()
},
```

## 五、功能模块与优先级

### P0 — MVP（第 1 周）

必须完成，最小可用版本：

| 功能 | 说明 | 技术要点 |
|------|------|----------|
| 全屏 Canvas 画布 | 黑色背景，占满整个屏幕 | Canvas 2.0 + 设备像素比适配 |
| 多指触控检测 | 检测所有手指按下/移动/抬起 | touchstart/move/end + e.touches |
| 手指圆圈绘制 | 每个手指显示不同颜色的圆圈 | Canvas 绑定 + 颜色数组 |
| 随机选人 | 所有手指稳定后随机选中一个 | Fisher-Yates 洗牌算法 |
| 选中动画 | 选中手指放大高亮，未选中淡出 | requestAnimationFrame 动画 |

### P1 — 增强功能（第 2 周）

提升体验和玩法丰富度：

| 功能 | 说明 |
|------|------|
| 选多人模式 | 设置选中 2~N 个人 |
| 分组模式 | 自动分成 2~N 组 |
| 音效反馈 | 按下音效 + 选中音效 |
| 设置页面 | 选择模式和参数 |
| 触觉反馈 | wx.vibrateShort() 震动 |

### P2 — 差异化功能（第 3 周+）

打造独特卖点：

| 功能 | 说明 |
|------|------|
| 主题皮肤 | 多套视觉主题（霓虹、春节、极简等） |
| 分享到群 | 生成结果卡片，分享到微信群 |
| 历史记录 | 本地存储每次选择结果 |
| 自定义场景 | 预设"谁买单""谁唱歌"等场景文案 |

```
┌─────────────────────────┐
│                         │
│    （全屏黑色背景）       │

│       ○ (蓝)            │

│                ○ (紫)   │
│                         │
│  ┌───┐                  │
│  │ ⚙ │  （设置按钮）     │
│  └───┘                  │
│                         │

│  （提示文字，手指放上后隐藏）│
│                         │
└─────────────────────────┘
```

### 交互流程

```
无手指 → 显示提示文字"将手指放在屏幕上"
         ↓
手指按下 → 每个手指位置出现彩色圆圈（渐入动画）
         ↓
稳定 1.5 秒 → 开始选择动画（圆圈闪烁/旋转）
         ↓
选中 → 被选中手指放大高亮 + 音效 + 震动
       未选中手指淡出消失
         ↓
所有手指离开 → 回到初始状态
```

## 八、上线发布流程

```
第一步：开发调试
  └─ 在微信开发者工具中编写代码、实时预览
  └─ 使用模拟器 + 真机预览调试

第二步：体验版测试
  └─ 点击开发者工具"上传"按钮，上传代码到微信后台
  └─ 在管理后台设置体验版，邀请测试人员扫码体验

第三步：提交审核
  └─ 在管理后台点击"提交审核"
  └─ 填写版本描述、功能说明
  └─ 等待微信团队审核（通常 1~3 个工作日）

第四步：完成备案
  └─ 在管理后台提交备案申请（建议与开发同步进行）
  └─ 等待备案审核通过

第五步：正式发布
  └─ 审核通过 + 备案完成后，点击"发布"
  └─ 小程序正式上线，所有微信用户可搜索使用
```
